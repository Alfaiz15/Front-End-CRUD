## 2. ğŸ—ƒï¸ Aplikasi Inti: SDMS (Manajemen Data Operasional CRUD)

Komponen ini merupakan inti *back-end* yang bertanggung jawab atas alur kerja data. Tujuannya adalah memastikan data operasional dikelola secara efisien dan konsisten.

### Fungsionalitas CRUD:

Aplikasi ini mengimplementasikan kerangka kerja **CRUD (Create, Read, Update, Delete)** untuk mengelola siklus hidup data:

* **CREATE (Data Ingestion):** Memungkinkan input data baru dengan validasi dasar, memastikan data masuk ke sistem sesuai standar **kualitas data minimum**.
* **READ (Data Retrieval):** Mengambil data secara *real-time* untuk kebutuhan **pelaporan** dan audit.
* **UPDATE (Data Maintenance):** Mendukung pemeliharaan dan **integritas data** dengan memungkinkan modifikasi data yang ada.
* **DELETE (Data Pruning):** Digunakan untuk **tata kelola data** dan menghapus data yang tidak lagi relevan.
